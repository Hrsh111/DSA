Sets vs Maps in C++

Sets:
1. Definition:
   - A `set` is a collection of unique elements.
   - It can be ordered (`std::set`) or unordered (`std::unordered_set`).

2. Initialization:
   // Ordered set
   std::set<int> ordered_set;

   // Unordered set
   std::unordered_set<int> unordered_set;

   // Initialization with values
   std::set<int> s = {1, 2, 3, 4};
   std::unordered_set<int> us = {1, 2, 3, 4};

3. Use Cases:
   - Storing unique elements.
   - Quickly checking the existence of an element.
   - Operations like union, intersection, and difference of sets.

4. Popular Problems:
   - Remove duplicates from an array.
   - Find the first unique element.
   - Check for a subset in a collection.

5. Time Complexities:
   - `std::set`: O(log N) for insertion, deletion, and search (uses balanced binary search tree).
   - `std::unordered_set`: O(1) on average for insertion, deletion, and search (uses hash table).

---

Maps:
1. Definition:
   - A `map` is a collection of key-value pairs.
   - It can be ordered (`std::map`) or unordered (`std::unordered_map`).

2. Initialization:
   // Ordered map
   std::map<int, int> ordered_map;

   // Unordered map
   std::unordered_map<int, int> unordered_map;

   // Initialization with values
   std::map<int, int> m = {{1, 100}, {2, 200}, {3, 300}};
   std::unordered_map<int, int> um = {{1, 100}, {2, 200}, {3, 300}};

3. Use Cases:
   - Storing data in key-value pairs.
   - Quickly looking up values using keys.
   - Counting frequencies of elements.

4. Popular Problems:
   - Frequency counting.
   - Finding the first non-repeating character.
   - Caching recently used data.

5. Time Complexities:
   - `std::map`: O(log N) for insertion, deletion, and search (uses balanced binary search tree).
   - `std::unordered_map`: O(1) on average for insertion, deletion, and search (uses hash table).

---

Key Differences:
1. Sets store only keys, while maps store key-value pairs.
2. Ordered (`std::set`/`std::map`) maintains elements in sorted order; unordered does not.
3. Use sets when you need only unique elements, and use maps when you need a key-value relationship.

/*
Understanding std::unordered_map in C++:

1. Key-Value Pair:
   - A std::unordered_map stores elements as key-value pairs.
   - Each element is represented as std::pair<KeyType, ValueType>.
     - first: Represents the key.
     - second: Represents the value associated with the key.

2. Example:
   unordered_map<int, int> map = {{1, 3}, {2, 2}, {3, 1}};
   - Here:
     - 1 is the key, 3 is the value.
     - 2 is the key, 2 is the value.
     - 3 is the key, 1 is the value.

3. Accessing Keys and Values:
   - You can iterate over the map using a range-based for loop:
     for (auto& entry : map) {
         cout << "Key: " << entry.first << ", Value: " << entry.second << endl;
     }
   - Output for the above map:
     Key: 1, Value: 3
     Key: 2, Value: 2
     Key: 3, Value: 1

4. Practical Usage:
   - first: Use entry.first to access the key.
   - second: Use entry.second to access the value (e.g., a count or other data).
*/
